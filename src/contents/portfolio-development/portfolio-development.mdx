제 포트폴리오 웹사이트를 만들면서 겪은 경험과 배운 점들을 공유합니다.

## 기술 스택 선정

포트폴리오 제작에 사용한 기술 스택입니다:

- **Next.js 16**: App Router, Server Components
- **React 19**: use 훅, forwardRef 제거
- **Three.js**: 3D 그래픽 렌더링
- **GSAP**: 부드러운 애니메이션
- **Tailwind CSS v4**: 빠르고 효율적인 스타일링

## 3D 피아노 히어로 섹션

가장 도전적이었던 부분은 3D 피아노를 구현하는 것이었습니다.

### React Three Fiber 활용

Three.js를 React에서 사용하기 위해 React Three Fiber를 선택했습니다:

```tsx
import { Canvas } from '@react-three/fiber';
import { OrbitControls, PerspectiveCamera } from '@react-three/drei';

export default function PianoScene() {
  return (
    <Canvas>
      <PerspectiveCamera makeDefault position={[0, 5, 10]} />
      <OrbitControls />
      <ambientLight intensity={0.5} />
      <directionalLight position={[10, 10, 5]} />
      <Piano />
    </Canvas>
  );
}
```

### 인터랙션 구현

피아노 건반을 클릭하면 소리가 나도록 구현했습니다:

```tsx
import Soundfont from 'soundfont-player';
import { type ThreeEvent } from '@react-three/fiber';

function PianoKey({ note }: { note: string }) {
  const meshRef = useRef<THREE.Mesh>(null);

  const handleClick = (e: ThreeEvent<MouseEvent>) => {
    e.stopPropagation();
    playNote(note);
    animateKey(meshRef.current);
  };

  return (
    <mesh ref={meshRef} onClick={handleClick}>
      <boxGeometry args={[1, 0.2, 5]} />
      <meshStandardMaterial color="white" />
    </mesh>
  );
}
```

## GSAP 스크롤 애니메이션

스크롤에 반응하는 애니메이션을 GSAP ScrollTrigger로 구현했습니다:

```tsx
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
import { useEffect, useRef } from 'react';

export default function AnimatedSection() {
  const sectionRef = useRef<HTMLElement>(null);

  useEffect(() => {
    if (!sectionRef.current) return;

    gsap.registerPlugin(ScrollTrigger);

    gsap.from(sectionRef.current, {
      opacity: 0,
      y: 100,
      scrollTrigger: {
        trigger: sectionRef.current,
        start: 'top 80%',
        end: 'top 20%',
        scrub: true,
      },
    });
  }, []);

  return <section ref={sectionRef}>Content</section>;
}
```

### Three.js 성능 개선

- **LOD (Level of Detail)**: 거리에 따라 모델 디테일 조정
- **Instancing**: 같은 모델 여러 개 효율적으로 렌더링
- **그림자 최적화**: 불필요한 그림자 비활성화

## 배운 점

1. **3D는 어렵지만 재밌다**: Three.js를 처음 배웠지만, 결과물을 보니 뿌듯했습니다
2. **성능이 중요하다**: 3D 렌더링은 성능에 민감하므로 최적화가 필수입니다
3. **타입 안정성**: TypeScript strict 모드로 개발하니 버그가 크게 줄었습니다
4. **애니메이션은 디테일**: 작은 애니메이션 하나가 사용자 경험을 크게 바꿉니다
